= Foreman on OpenShift with Helm

A demonstration of deploying Foreman 3.13 lifecycle management platform on OpenShift using custom Helm charts integrated with the RHDP Zerotouch platform.

== Overview

This repository contains a production-ready Helm chart for Foreman along with comprehensive documentation explaining the architecture, deployment patterns, and AgnosticD workload configuration.

== What's Included

=== Helm Chart (`foreman-chart/`)

Custom Helm chart deploying Foreman with:

* **Multi-tier architecture**: Application pod (Rails + Sidekiq), PostgreSQL StatefulSet, dual Redis instances
* **RHDP integration**: GUID/domain templating, common_password support, Route creation
* **Production-ready**: Resource limits, health checks, initContainer dependency ordering
* **OpenShift native**: RHEL 9 base images, NetworkPolicy support, SCC compatible

=== Configuration (`config/`)

* **`helm-charts.yaml`**: Deployment manifest for AgnosticD's `ocp4_workload_helm_from_content_repo`
* **`firewall.yaml`**: NetworkPolicy rules for pod-to-pod and external communication
* **`instances.yaml`**: Empty (Helm-only deployment, no VMs)

=== Documentation (`content/`)

Comprehensive technical blog covering:

* Foreman's Kubernetes architecture and component relationships
* Helm chart integration patterns for Showroom labs
* AgnosticD workload configuration and variable substitution
* NetworkPolicy setup for lab environments
* Troubleshooting and verification procedures

== Target Audience

RHDP team members and lab developers familiar with OpenShift and Helm who want to understand how to integrate custom Helm charts into zerotouch lab deployments.

== Deployment

This repository is designed to be provisioned via AgnosticD's zerotouch workflow:

1. **Content repo**: Catalog item references this repo via `ocp4_workload_showroom_content_git_repo`
2. **Helm workload**: `ocp4_workload_helm_from_content_repo` reads `config/helm-charts.yaml`
3. **Chart deployment**: Helm renders `foreman-chart/` with GUID and domain substitution
4. **Route creation**: AgnosticD creates OpenShift Route for external access
5. **NetworkPolicy**: Firewall rules applied in `post_software` phase

== Access

After deployment:

* **Web UI**: `https://foreman-{guid}.{domain}/`
* **API**: `https://foreman-{guid}.{domain}/api/ping`
* **Credentials**: Username `admin` / Password from `{{ common_password }}`

== Architecture Highlights

* **Single-pod deployment** with 3 containers: foreman-app, orchestrator, worker
* **StatefulSet for PostgreSQL** (10Gi PVC) and Redis-tasks (2Gi AOF persistence)
* **Deployment for Redis-cache** (256Mi in-memory, LRU eviction)
* **InitContainers** enforce startup ordering: wait-postgres → db-init → main containers
* **Service discovery** via Kubernetes ClusterIP services and DNS
